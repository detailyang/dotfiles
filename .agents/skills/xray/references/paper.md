# Paper Mode

适用场景：用户给出 arXiv 编号、论文 URL、论文 PDF、本地 PDF、论文标题，或者明确要求解读一篇研究论文。

目标只有两个：

1. **论文说了什么**：问题 -> 视角 -> 结果
2. **对我意味着什么**：它改变了什么认知或决策方式

## 红线

每次输出前都过一遍：

1. **口语检验**
   - 这像你会跟朋友讲的话吗
   - 不像就重写
2. **零术语优先**
   - 先用人话落地，再补术语名
3. **短词优先**
   - 能短就短，能具体就具体
4. **一句一事**
   - 每句只推进一步
5. **诚实**
   - 看不懂就说看不懂
   - 有硬伤就说硬伤

## 获取内容

### 输入归一化

按下面规则先把输入变成可读取的正文：

| 用户输入 | 处理方式 |
| --- | --- |
| `2601.01290` 或 `arxiv:2601.01290` | 转成 `https://arxiv.org/html/2601.01290` |
| `https://arxiv.org/abs/...` 或 `/pdf/...` | 优先改写成 `/html/...` |
| `https://arxiv.org/html/...` | 直接读取 |
| 在线 PDF | 读取 PDF |
| 本地 PDF 或本地文件 | 读取文件 |
| 只有论文标题 | 先搜索，再锁定论文 |

尽量拿到：

- 标题
- 作者
- 摘要
- 核心方法
- 关键实验
- 结论与局限

## 执行步骤

### 1. 定位问题

找到那个真实困境：为什么这件事之前做不到、解释不通或代价太高。

不要写“本文提出了一种新的框架”。要写“以前哪里卡住了”。

### 2. 费曼讲解

用三句话讲清：

- **问题**：作者要解决什么
- **视角**：作者换了什么角度或做了什么东西
- **结果**：得到了什么，哪些是证据，哪些只是作者推测

要求：

- 先人话，后术语
- 承重概念必须场景化，不要括号注释式解释
- 最后压成一句电梯话术，不准充满术语

### 3. 餐巾纸图

用一张 ASCII 图画出核心机制、流程或对比关系。目标是让读者在不读长文的前提下抓到论文结构。

### 4. 核心概念

挑 1 到 3 个最承重的概念逐个拆开。每个概念回答：

- **一句话**：这是什么，干什么用
- **类比或例子**：让没接触过的人也能跟上
- **为什么重要**：少了它，论文哪条逻辑链会塌

### 5. 洞见

整篇论文真正的“哦，原来……”通常只有一个。找到它，并说明它改变了哪个默认认知。

### 6. 博导审稿

换成带了二十年学生的博导视角，判断这篇论文值不值得认真对待：

- **选题眼光**
- **方法成熟度**
- **实验诚意**
- **写作功力**
- **判决**
  - `strong accept`
  - `weak accept`
  - `borderline`
  - `weak reject`
  - `strong reject`

### 7. 对我意味着什么

先用一句话回答：这篇论文可能改变我的什么。

然后从三个角度试探：

- **迁移**：有没有机制能移植到别的场景
- **混搭**：能不能和现有方法组合
- **反转**：它是否推翻了我的某个默认假设

命中的点才展开成认知卡片；没命中就写 `delta ≈ 0`。

### 8. 认知卡片

卡片必须用 ASCII 表达结构，不是给文字画边框。

合格卡片至少呈现一种关系：

- 分叉
- 层级
- 阈值
- 对比
- 拉扯
- 翻转

卡片下方附一句能脱离论文上下文单独成立的启发。

## 输出结构

```markdown
# {论文标题}

**作者**: {作者}
**来源**: {会议、期刊或 arXiv}
**链接**: {链接或文件路径}

## 问题

## 论文说了什么

### 问题

### 视角

### 结果

### 一句话总结

### 餐巾纸图

## 核心概念

## 洞见

## 博导审稿

## 对我意味着什么
```

## 质量标准

- 读者不懂该领域也能跟上主线
- “结果”里要区分证据和推测
- 审稿部分要真的下判断，不能模糊
- 如果论文本身信息不足，不要脑补作者没写的内容
