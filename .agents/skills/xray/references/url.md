# URL Mode

适用场景：用户提供网页、文章、博客、文档、论坛帖子或产品页，希望提取、分析、总结或理解其内容。

目标不是做摘要，而是通过三轮压缩，把“页面写了什么”变成“这对我意味着什么”。

## 核心原则

- 同一页面里既有结构也有噪声，分析深度取决于你投入多少认知算力
- 不满足于标题改写
- 不要把“作者说了什么”误当成“作者真正回应了什么”

## 执行步骤

### 1. 获取正文

用当前可用的网页读取工具抓取正文。优先拿结构化文本，避免导航栏、评论噪声和无关脚本。

如果页面过长：

- 优先锁定正文、摘要、关键章节
- 技术文档保留标题层级和代码示例上下文
- 新闻或博客保留时间、作者、引用对象

### 2. 三轮认知压缩

#### 第一轮：骨架扫描

回答“这个页面在说什么”。

- **核心主题**：页面试图传达什么
- **核心价值**：读者能获得什么
- **内容结构**：有哪些主要部分
- **内容类型**：教程、新闻、博客、文档、论坛、产品页或其他

#### 第二轮：血肉解剖

回答“它是怎么说的”。

- **真问题**：作者真正回应的困境是什么
- **真回答**：去掉就全文坍塌的核心主张
- **关键论据**：最多 3 个
- **隐含假设**：什么被默认为真
- **边界**：在哪些条件下会失效
- **论证拓扑图**：用 ASCII 画出从真问题到真回答的路径

如果“真问题”只是标题改写，说明还没挖到位，继续往下找。

#### 第三轮：灵魂提取

回答“对我有什么用”。

- **认知碰撞**：它改变了什么判断、工具或视角
- **知识连接**：它和已有知识的交叉点
- **延伸资源**：继续深挖可看的方向
- **一句话总结**：压成一句

### 3. 认知卡片

只有在确实有碰撞时才生成 ASCII 卡片。

卡片要让结构本身说话，能直接看出分叉、层级、阈值、翻转或张力。不要把三句话塞进一个框里就算卡片。

如果没有碰撞，直接写：`无新的认知碰撞`。

## 输出结构

```markdown
# {页面标题}

## 一句话总结

## 第一轮：骨架扫描

### 核心主题

### 核心价值

### 内容结构

### 内容类型

## 第二轮：血肉解剖

### 真问题

### 真回答

### 论证拓扑图

### 关键论据

### 隐含假设

### 边界

## 第三轮：灵魂提取

### 认知碰撞

### 知识连接

### 延伸资源
```

## 保存策略

- 默认直接在对话中输出
- 只有用户明确要求“保存报告 / 落盘 / 生成 md 文件”时，才写文件
- 如果需要写文件，文件名简短可读，保留来源 URL 和日期

## 质量标准

- 真问题必须比标题更深一层
- 延伸资源要服务于继续理解，不是随便贴链接
- “认知碰撞”必须是你的判断真的被改变了；否则就诚实写没有
