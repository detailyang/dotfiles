# RESOURCES.md Format

`RESOURCES.md` is the curated set of trusted sources for a learning topic. Use it to avoid re-discovering sources and to keep lessons grounded.

## Structure

```md
# {Topic} Resources

## Knowledge

- [Resource title](https://example.com)
  What it covers. Use for: when to consult it.

## Wisdom / Communities

- [Community name](https://example.com)
  Why it is high-signal. Use for: real-world feedback or judgment.

## Gaps

- Missing source for {area}
```

## Rules

- Prefer primary sources, official docs, recognized experts, peer-reviewed work, and well-moderated communities.
- Annotate every entry with what it is useful for.
- Separate knowledge sources from communities.
- Surface gaps explicitly.
- Prune shallow, wrong, or off-mission resources.
- Record if the user does not want community recommendations.
