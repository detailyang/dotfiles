# MISSION.md Format

`MISSION.md` captures why the user is learning this topic. Teaching choices should trace back to it.

## Template

```md
# Mission: {Topic}

## Why

{1-3 sentences describing the concrete real-world outcome the user wants.}

## Success looks like

- {observable capability}
- {observable capability}

## Constraints

- {time, budget, prior knowledge, preferences, commitments}

## Out of scope

- {adjacent topics not being pursued now}
```

## Rules

- One mission per learning workspace.
- Prefer concrete outcomes over abstract curiosity.
- Push back on vague missions before writing a long lesson.
- Revise when reality shifts, with user confirmation.
- Keep it short enough to remain a compass.
