# GLOSSARY.md Format

`GLOSSARY.md` is the canonical language for a learning workspace. Lessons, exercises, and learning records should use its terminology.

Building the glossary is part of learning: compressing a concept into a tight definition is evidence of understanding.

## Structure

```md
# {Topic} Glossary

{One or two sentence description of the topic.}

## Terms

**Term**:
One or two sentence definition.
_Avoid_: loose synonym, misleading alias
```

## Rules

- Add a term only when the user understands it well enough to use it correctly.
- Pick one canonical term when several words compete.
- Keep definitions tight.
- Use glossary terms inside later definitions.
- Group terms under subheadings when natural clusters emerge.
- Flag ambiguity explicitly: "In this workspace, X means Y."
- Revise stale definitions as understanding deepens.
