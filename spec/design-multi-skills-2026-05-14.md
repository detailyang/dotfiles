# Design: multi-skills Extension

Generated by /think on 2026-05-14
Status: Draft

## Problem

用户经常需要同时激活多个 skills 并结合自定义指令发送给 agent。目前只能通过 `/skill:name` 逐个加载 skill，无法批量选择，也无法在同一个交互流中附带自定义 prompt。需要一个 `/multi-skills` 命令，提供两阶段交互——先多选 skills，再输入文本——最终将选中 skills 的完整内容和用户输入组装为结构化 prompt 发给 agent。

## Out of Scope

- 不修改 skill 的加载/发现机制
- 不持久化用户的 skill 选择偏好（每次都是临时选择）
- 不修改 system prompt 注入方式（通过 `sendUserMessage` 而非 `before_agent_start`）
- 不处理 skill 内容过大导致 token 超限的问题（由 agent 上层处理）
- 不做 skill 列表的 fuzzy filter 搜索（v1）

## Constraints

1. 必须是 extension（`extensions/multi-skills/index.ts`），不是 skill
2. 使用 `pi.sendUserMessage()` 发送最终 prompt，不注入 system prompt
3. 读取 SKILL.md 完整内容放入 `<content>` 标签
4. 两阶段交互，无焦点切换：阶段1 多选 overlay → 阶段2 `ctx.ui.input()`
5. overlay 占 90% 宽高，支持十几个 skills 滚动
6. 遵循项目现有模式：`@earendil-works/pi-coding-agent` + `@earendil-works/pi-tui` 组件
7. 单文件 extension，无额外依赖

## Assumptions

1. `pi.getCommands()` 在 command handler 调用时能返回已加载的 skill commands（`source === "skill"`）
2. skill command 的 `sourceInfo.path` 指向 SKILL.md 文件，可通过 `readFileSync` 读取
3. `ctx.ui.input()` 在 overlay 关闭后可以正常弹出（无 UI 冲突）
4. `SelectList` 的 `handleInput` 可能内部消费 space 键，因此不可靠，需要自定义组件

## Approach

### 阶段 1：MultiSelectList 组件

自定义组件，不依赖 SelectList，实现 `Component` 接口：

**状态：**
- `items: Array<{ value: string; label: string; description: string; path: string }>`
- `checked: Set<string>` — 已选中
- `selectedIndex: number` — 光标位置
- `scrollOffset: number` — 滚动偏移
- `maxVisible: number` — 可见行数（15）

**render(width)：**
```
  [x] brave-search   Web search and content extraction via Brave Search API
  [ ] pdf-tools      Extracts text and tables from PDF files
▸ [x] tmux           Tmux session management and pane control
  [ ] code-review    Automated code review with style checking
```
- checkbox `▸` 标记当前光标行（accent 色）
- `[x]` accent+bold，`[ ]` dim 色
- label 部分 muted 色（光标行 accent 色）
- description 部分 dim 色
- 使用 `truncateToWidth` 确保行宽

**handleInput(data)：**
- `Key.up` / `Key.down`：移动 selectedIndex，调整 scrollOffset
- `Key.space`：toggle checked set 中当前 item
- 不响应其他键

**外部行为：**
- `Key.enter`：在外层 handleInput 中拦截，调用 `done(checked)`
- `Key.escape`：在外层 handleInput 中拦截，调用 `done(null)`

### 阶段 2：输入框

直接调用 `ctx.ui.input("Prompt for selected skills:", "")`：
- 内置组件，IME 开箱即用
- 返回 `string | undefined`（undefined = 用户取消）

### 组装 XML prompt

```typescript
function buildPrompt(
  selectedSkills: Array<{ name: string; description: string; content: string }>,
  userInput: string,
): string {
  let xml = "<available_skills>\n";
  for (const skill of selectedSkills) {
    xml += `<skill name="${skill.name}">\n`;
    xml += `<description>${skill.description}</description>\n`;
    xml += `<content>\n${skill.content}\n</content>\n`;
    xml += `</skill>\n`;
  }
  xml += "</available_skills>";
  if (userInput.trim()) {
    xml += "\n\n" + userInput.trim();
  }
  return xml;
}
```

## Key Decisions

1. **自定义 MultiSelectList vs hack SelectList**：选择自定义。SelectList 无 `setItems()` API，space 键行为不可控。自定义组件 ~50 行 render+input，行为完全可控，视觉行内 `[x]`/`[ ]` 清晰。只要 SelectList 不增加 multiSelect 支持，这个决定成立。

2. **两阶段 vs 单屏 overlay**：选择两阶段。单屏需要 Tab 焦点切换 + Focusable 接口实现 + IME 传播，复杂度高且易出 bug。两阶段每步组件单一职责，`ctx.ui.input()` 内置 IME 支持开箱即用。

3. **sendUserMessage vs before_agent_start**：选择 `sendUserMessage`。prompt 以用户消息形式出现，agent 立即处理。

4. **SKILL.md 读取路径**：`pi.getCommands()` → `sourceInfo.path` → `readFileSync`。如果读取失败，content 标记为 `[Error: unable to read ${path}: ${error.message}]`。

5. **无 skills 时直接返回**：`ctx.ui.notify("No skills available", "warning")`，不弹 overlay。

## Failure Modes & Mitigations

| Failure Mode | Mitigation |
|---|---|
| 无 skills 可用 | notify warning 并返回 |
| SKILL.md 读取失败 | content 写入错误标记，不阻塞 |
| 阶段1 选 0 个 skill 就按 Enter | 跳过阶段2，notify "No skills selected" 并返回 |
| 阶段2 用户取消（Esc） | 丢弃选择，不发送任何消息 |
| 阶段2 用户输入空文本 | 仍发送 `<available_skills>` XML，无尾部文本 |
| skill 内容过大 | 不裁剪，由 agent token 管理处理 |

## Test Coverage

| 场景 | 验证方式 |
|---|---|
| Happy path：选2个 skills + 输入文本 → 正确 XML | 手动测试，检查 sendUserMessage 内容 |
| 无 skills | 删除 skill 目录后运行，应显示 warning |
| 0 skills 选中 → Enter | 应 notify "No skills selected" 并返回 |
| N skills 选中 + 阶段2 取消 | 不发送任何消息 |
| N skills 选中 + 空输入 | 发送纯 XML 无尾部文本 |
| Esc 在阶段1 | 关闭 overlay，不弹 input |
| SKILL.md 读取失败 | content 包含错误标记 |
| 15+ skills 滚动 | 验证 ↑↓ 滚动 + checkbox 正确 |

⚠️ IME 输入场景需真实终端手动验证（`ctx.ui.input()` 内置支持，风险低）。

## Rollback Plan

删除 `extensions/multi-skills/index.ts` 即可。无持久化、无外部依赖、不修改其他文件。

## Success Criteria

1. `/multi-skills` 命令弹出多选 overlay，列出所有已加载 skills
2. Space 键 toggle 选中状态，视觉行内 `[x]`/`[ ]` 反馈
3. Enter 确认后弹出 `ctx.ui.input()` 获取用户文本
4. 最终 prompt 包含选中 skills 的完整 SKILL.md 内容 + 用户输入
5. overlay 占 90% 屏幕宽高，15+ skills 可滚动
6. Esc 任意阶段取消，不发送任何内容

## Implementation Steps

### Step 1: 创建 extension 骨架 + 无 skill 检测

文件：`pi/extensions/multi-skills/index.ts`

```typescript
import type { ExtensionAPI, SlashCommandInfo } from "@earendil-works/pi-coding-agent";
import { readFileSync } from "node:fs";

interface SkillItem {
  value: string;     // skill name
  label: string;     // display name
  description: string;
  path: string;      // SKILL.md full path
}

function getSkillItems(pi: ExtensionAPI): SkillItem[] {
  return pi
    .getCommands()
    .filter((cmd: SlashCommandInfo) => cmd.source === "skill")
    .map((cmd: SlashCommandInfo) => ({
      value: cmd.name,
      label: cmd.name,
      description: cmd.description ?? "",
      path: cmd.sourceInfo.path,
    }));
}

export default function multiSkillsExtension(pi: ExtensionAPI) {
  pi.registerCommand("multi-skills", {
    description: "Select multiple skills and compose a prompt",
    handler: async (_args, ctx) => {
      const items = getSkillItems(pi);
      if (items.length === 0) {
        ctx.ui.notify("No skills available", "warning");
        return;
      }
      // TODO: Step 2 - overlay
    },
  });
}
```

验证：`/multi-skills` 在无 skill 时显示 warning。

### Step 2: 实现 MultiSelectList 组件

同文件内定义 class/函数：

```typescript
import { matchesKey, Key, truncateToWidth, visibleWidth, type Component } from "@earendil-works/pi-tui";

interface MultiSelectItem {
  value: string;
  label: string;
  description: string;
  path: string;
}

class MultiSelectList implements Component {
  private items: MultiSelectItem[];
  private checked: Set<string> = new Set();
  private selectedIndex = 0;
  private scrollOffset = 0;
  private maxVisible: number;
  private theme: ThemeColors;
  private cachedWidth?: number;
  private cachedLines?: string[];

  constructor(items: MultiSelectItem[], maxVisible: number, theme: ThemeColors) {
    this.items = items;
    this.maxVisible = maxVisible;
    this.theme = theme;
  }

  toggleCurrent(): void { /* toggle checked for items[selectedIndex] */ }
  getChecked(): Set<string> { return this.checked; }
  getSelectedPath(): string { return this.items[this.selectedIndex].path; }

  handleInput(data: string): void {
    if (matchesKey(data, Key.up)) { /* move up, adjust scroll */ }
    else if (matchesKey(data, Key.down)) { /* move down, adjust scroll */ }
    else if (matchesKey(data, Key.space)) { this.toggleCurrent(); }
  }

  invalidate(): void { this.cachedWidth = undefined; this.cachedLines = undefined; }

  render(width: number): string[] {
    // 计算可见窗口，每行格式：
    //   [x] skill-name   description
    // 或
    // ▸ [x] skill-name   description  (当前光标行)
  }
}
```

- `maxVisible` 固定为 15
- checkbox 颜色：选中 accent+bold，未选中 dim
- 光标行前缀 `▸` accent 色
- label 列宽固定 20 字符，description 占剩余空间
- 滚动：当 selectedIndex < scrollOffset 或 >= scrollOffset + maxVisible 时调整

验证：在 overlay 中渲染列表，↑↓ 移动光标，space toggle，滚动正常。

### Step 3: 阶段1 overlay 整合

在 command handler 中：

```typescript
const checked = await ctx.ui.custom<Set<string> | null>((tui, theme, _kb, done) => {
  const container = new Container();
  // 顶部 DynamicBorder (accent)
  // 标题 Text: "π Multi-Skills (0 selected)" — accent+bold
  // 分隔线 DynamicBorder (borderMuted)
  // MultiSelectList 实例
  // 分隔线 DynamicBorder (borderMuted)
  // 帮助 Text: "↑↓ navigate • space toggle • enter confirm • esc cancel"
  // 底部 DynamicBorder (accent)

  const selectList = new MultiSelectList(items, 15, theme);

  return {
    render(w) { return container.render(w); },
    invalidate() { container.invalidate(); },
    handleInput(data) {
      if (matchesKey(data, Key.escape)) { done(null); return; }
      if (matchesKey(data, Key.enter)) { done(selectList.getChecked()); return; }
      selectList.handleInput(data);
      // 更新标题中的 "(N selected)"
      tui.requestRender();
    },
  };
}, { overlay: true, overlayOptions: { width: "90%", maxHeight: "90%" } });
```

标题动态更新选中数量：维护 `titleText.setText(...)` 每次 toggle 后调用。

验证：完整阶段1 交互——选择 skills，Enter 返回 Set，Esc 返回 null。

### Step 4: 阶段2 输入 + XML 组装 + 发送

```typescript
// 阶段1 返回后
if (!checked || checked.size === 0) {
  if (checked && checked.size === 0) {
    ctx.ui.notify("No skills selected", "warning");
  }
  return;
}

// 阶段2
const userInput = await ctx.ui.input("Prompt for selected skills:", "");

// 读取选中 skills 的 SKILL.md
const selectedItems = items.filter(item => checked.has(item.value));
const skills = selectedItems.map(item => {
  let content: string;
  try {
    content = readFileSync(item.path, "utf-8");
  } catch (e: any) {
    content = `[Error: unable to read ${item.path}: ${e.message}]`;
  }
  return { name: item.value, description: item.description, content };
});

// 组装 XML
const prompt = buildPrompt(skills, userInput ?? "");

pi.sendUserMessage(prompt);
```

`buildPrompt` 函数按 Approach 中定义的格式拼接 XML。

验证：选中 2 个 skills + 输入文本 → 检查 agent 收到消息的 XML 格式和内容完整性。

### Step 5: 边缘情况和视觉 polish

1. 阶段2 `userInput === undefined`（用户 Esc 取消）：不发送消息，直接返回
2. `userInput === ""`（空输入）：发送纯 XML 无尾部文本
3. description 为 undefined 的 skill 渲染为空字符串
4. MultiSelectList 的 scrollOffset 边界：clamp 在 `[0, max(0, items.length - maxVisible)]`
5. 选中数量为 0 时标题显示 "0 selected"，Enter 后走 no-skills-selected 分支

验证：全部 Test Coverage 表中场景逐一通过。
